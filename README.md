# 🫀 Цифровой двойник человека (Digital Twin)

Проект создания модульной, многоагентной симуляционной системы человеческого организма. Система моделирует физиологические процессы на макро-уровне, позволяя предсказывать реакции организма на внешние воздействия (питание, физические нагрузки, лекарственные препараты, стресс) и отслеживать долгосрочные адаптации (например, развитие резистентности к инсулину).

## 🏗 Архитектура

Проект построен на базе **3-уровневой мультиагентной архитектуры**, обеспечивающей строгую изоляцию зон ответственности и высокую масштабируемость:

1. **Уровень 0: Стратегический оркестратор («Мозг/Врач»)**
   - Хранит паспорт пациента (`PatientPassport`).
   - Преобразует клинические метрики (возраст, вес, HOMA-IR, HbA1c) в физические коэффициенты для дифференциальных уравнений (калибровочный движок).
   - Принимает внешние воздействия (расписание питания, сон, лекарства) и транслирует их внутрь систем.

2. **Уровень 1: Супер-агенты систем организма (11 систем)**
   - Управляют гомеостазом в рамках своей системы (например, поддержание уровня глюкозы в крови).
   - Выступают оркестраторами для своих внутренних агентов (органов).

### 🧬 Планируемые системы и органы (Уровень 1 и Уровень 2)

В рамках проекта планируется полная симуляция следующих 11 систем организма:

1. **Энергетический гомеостаз (Глюкозо-инсулиновая система) — *Реализовано (MVP)***
   - *Органы:* Поджелудочная железа (`Pancreas_Agent`), Печень (`Liver_Agent`), Мышцы (`Muscle_Agent`), Жировая ткань (`Adipose_Agent`), ЖКТ (`Gut_Agent`), Почки (`Kidney_Agent`), Мозг (`Brain_Agent`).
2. **Сердечно-сосудистая система (Гемодинамика)**
   - *Органы:* Сердце (насосная функция, ЧСС, ударный объем), Сосудистое русло (тонус артерий/вен, сопротивление), Барорецепторы.
3. **Дыхательная система**
   - *Органы:* Легкие (альвеолярный газообмен O2/CO2), Диафрагма (частота и глубина дыхания), Дыхательный центр мозга.
4. **Нервная система (Вегетативная и ЦНС)**
   - *Компоненты:* Симпатический узел (адреналин/норадреналин, реакция "бей или беги"), Парасимпатический узел (вагус, отдых и пищеварение).
5. **Эндокринная система (Гормональный фон)**
   - *Органы:* Гипофиз, Гипоталамус, Надпочечники (кортизол/альдостерон), Щитовидная железа (Т3/Т4, базовый метаболизм).
6. **Выделительная система (Водно-электролитный баланс)**
   - *Органы:* Почки (нефроны: фильтрация, реабсорбция воды и натрия), Мочевой пузырь.
7. **Пищеварительная система (Нутрициология)**
   - *Органы:* Желудок, Тонкий кишечник (всасывание микро/макронутриентов), Толстый кишечник (микробиом).
8. **Иммунная система**
   - *Органы/Агенты:* Лейкоциты, Макрофаги, Лимфатические узлы, Селезенка, Тимус (симуляция воспаления).
9. **Опорно-двигательная система**
   - *Органы:* Скелетная мускулатура (потребление АТФ, лактат), Кости (кальциевый обмен).
10. **Репродуктивная система**
    - *Органы:* Половые железы (тестостерон/эстроген/прогестерон, влияние на анаболизм).
11. **Терморегуляция**
    - *Органы:* Кожа (потоотделение, вазодилатация), Бурый жир (термогенез), Гипоталамус (точка температуры).

3. **Уровень 2: Агенты-органы**
   - Узкоспециализированные микросервисы (как описано выше), вычисляющие дифференциальные уравнения для конкретных тканей и обменивающиеся метаболитами через общую среду (`BloodPool`).

## ⚙️ Ядро системы (Core)

В основе движка лежит:
*   **Tick-based Simulation Engine**: Механизм дискретного времени. Поддерживает работу агентов с разной частотой (органы могут «тикать» раз в 1 минуту, а агенты долгосрочной адаптации — раз в 24 часа).
*   **Zero-cost Message Bus**: Внутренняя шина событий (на базе `dataclasses`). Органы общаются между собой исключительно через публикацию и подписку на биологические события (например, `MealIntakeMsg`).
*   **Blood Pool**: Математическое ядро, решающее дифференциальные уравнения (ОДУ) изменения концентрации веществ (Глюкоза, Инсулин, Глюкагон) методом Эйлера на каждом тике симуляции.

## 🚀 Быстрый старт

### Требования
- Python 3.10+

### Установка
1. Клонируйте/откройте репозиторий.
2. Установите зависимости (если вы планируете использовать визуализацию и UI):
   ```bash
   pip install matplotlib numpy scipy streamlit plotly pandas
   ```

### Использование CLI (Сценарии)
Вы можете запустить симуляцию перорального глюкозотолерантного теста (ПГТТ) из командной строки. Скрипт проведёт симуляцию и сгенерирует график `.png`.
```bash
python3 scenarios/run_ogtt.py
```

### Запуск интерактивного GUI (Streamlit)
Для удобного взаимодействия с цифровым двойником, изменения параметров пациента и визуализации процессов в реальном времени, запустите веб-интерфейс:
```bash
python3 -m streamlit run app.py
```

## 🧪 Валидация и Тестирование

Математическая модель откалибрована так, чтобы удовлетворять строгим медицинским критериям. Вы можете прогнать автоматизированный Validation Suite:
```bash
python3 tests/test_validation.py
```
**Набор тестов проверяет:**
*   Способность «здорового» паспорта поддерживать гомеостаз (5.0 ммоль/л) натощак.
*   Возвращение глюкозы к норме (< 7.8 ммоль/л) через 2 часа после углеводной нагрузки 75г.
*   Демонстрацию гипергликемии у паспорта с Диабетом 2-го типа (учитывается пониженная чувствительность к инсулину и сниженная масса бета-клеток из-за глюкотоксичности при высоком HbA1c).
